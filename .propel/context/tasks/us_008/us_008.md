# User Story: US-008 - Patient Booking Confirmation

**Story ID:** US-008  
**Epic:** EP-001 (Patient Experience)  
**Created:** April 17, 2026  
**Status:** Draft  
**Priority:** High  
**Effort:** 3 story points  

---

## User Story

**As a** patient,  
**I want to** review and confirm my appointment booking,  
**So that** I can ensure all details are correct before finalizing.

---

## Acceptance Criteria

### Functional Requirements
- [ ] Display selected provider, date, time, and location
- [ ] Show appointment type and estimated duration
- [ ] Include patient contact information for verification
- [ ] Display insurance information if provided
- [ ] Show preparation instructions or requirements
- [ ] Provide clear confirm and cancel options
- [ ] Prevent booking if slot becomes unavailable

### Non-Functional Requirements
- [ ] Review loads instantly from cached selection
- [ ] Confirmation completes within 2 seconds
- [ ] Form accessible with keyboard navigation

### Edge Cases
- [ ] Slot no longer available during review
- [ ] Patient information incomplete
- [ ] Insurance validation fails
- [ ] Duplicate booking attempt

### Key UI Elements

- Patient info: Name, contact, insurance
- Action buttons: Large confirm (primary), cancel (secondary)

### Design Tokens
- Summary card: White background with blue accent (#3B6FB7)
- Confirm button: Full-width primary blue
- Warning text: Amber color (#C47A00)

### Responsive Behavior
- Desktop: Side-by-side summary and actions
