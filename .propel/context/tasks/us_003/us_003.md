# User Story: US-003 - Staff Appointment Scheduling

**Story ID:** US-003  
**Epic:** EP-002 (Staff Operations)  
**Created:** April 17, 2026  
**Status:** Draft  
**Priority:** High  
**Effort:** 10 story points  

---

## User Story

**As a** clinical scheduler,  
**I want to** triage and schedule patient appointments based on clinical urgency,  
**So that** patients receive appropriate care in optimal timeframes.

---

## Acceptance Criteria

### Functional Requirements
- [ ] Scheduler can view patient requests in priority queue
- [ ] Triage form captures chief complaint, urgency level, and appointment type
- [ ] System recommends providers and time slots based on rules
- [ ] Scheduler can override recommendations with clinical judgment
- [ ] Appointment booking includes patient clinical context
- [ ] System prevents double-booking and respects provider availability
- [ ] Booking confirmation sent to patient immediately

### Non-Functional Requirements
- [ ] Triage process completes within 2 minutes average
- [ ] Supports 50 concurrent schedulers
- [ ] Real-time availability updates
- [ ] 99.5% uptime for scheduling operations

### Edge Cases
- [ ] Urgent cases bypass standard availability rules
- [ ] Provider unavailable for recommended specialty
- [ ] Patient requires multiple appointment types
- [ ] Last-minute cancellations create availability gaps

## Visual Design Context

### Screen References
- SCR-010: Scheduler Dashboard (overview and queue)
- SCR-012: Appointment Triage & Booking (main workflow)
- SCR-013: Appointment Detail (Staff) (confirmation)

### Key UI Elements
- Triage panel: Chief complaint dropdown, urgency selector, appointment type
- Recommendation cards: Provider photo, availability, confidence score
- Booking modal: Patient details, appointment summary, override options
- Queue list: Patient name, wait time, urgency indicator

### Design Tokens
- Urgency colors: Routine (blue), Urgent (amber), Critical (red)
- Status indicators: Available (green), Booked (gray), Recommended (blue)
- Action hierarchy: Primary (Book), Secondary (Override), Tertiary (Skip)

### Responsive Behavior
- Desktop: Multi-panel layout with queue, triage, and calendar
- Tablet: Stacked panels with collapsible sections
- Mobile: Single-column with step-by-step wizard
