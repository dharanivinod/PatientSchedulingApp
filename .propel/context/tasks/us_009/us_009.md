# User Story: US-009 - Patient Booking Receipt

**Story ID:** US-009  
**Epic:** EP-001 (Patient Experience)  
**Created:** April 17, 2026  
**Status:** Draft  
**Priority:** High  
**Effort:** 2 story points  

---

## User Story

**As a** patient,  
**I want to** receive confirmation of my successful appointment booking,  
**So that** I have a record and know what to expect.

---

## Acceptance Criteria

### Functional Requirements
- [ ] Display booking confirmation with all appointment details
- [ ] Show next steps and preparation instructions
- [ ] Provide options to add to personal calendar
- [ ] Include reminder schedule information
- [ ] Offer ability to modify or cancel from confirmation
- [ ] Send confirmation via email and SMS automatically
- [ ] Display unique booking reference number

### Non-Functional Requirements
- [ ] Confirmation appears immediately after booking
- [ ] Notifications sent within 2 minutes
- [ ] Receipt accessible from appointment history

### Edge Cases
- [ ] Notification delivery fails
- [ ] Patient closes page before notifications sent
- [ ] Calendar integration not available
- [ ] Multiple bookings in same session
