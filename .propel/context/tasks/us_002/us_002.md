# User Story: US-002 - Patient Appointment Management

**Story ID:** US-002  
**Epic:** EP-001 (Patient Experience)  
**Created:** April 17, 2026  
**Status:** Draft  
**Priority:** High  
**Effort:** 6 story points  

---

## User Story

**As a** patient,  
**I want to** view and manage my upcoming and past appointments,  
**So that** I can track my healthcare schedule and make changes as needed.

---

## Acceptance Criteria

### Functional Requirements
- [ ] Patient can view calendar of all appointments (upcoming 12 months, past 12 months)
- [ ] Appointments display with status chips (confirmed, rescheduled, cancelled, no-show)
- [ ] Patient can reschedule appointments up to 24 hours before
- [ ] Patient can cancel appointments with reason selection
- [ ] System shows alternative slots when rescheduling
- [ ] Past appointments include clinical notes and visit summaries
- [ ] Appointments sync with personal calendar (Google, Apple, Outlook)

### Non-Functional Requirements
- [ ] Calendar loads within 2 seconds
- [ ] Supports 1000+ appointments per patient
- [ ] Real-time updates for status changes
- [ ] Accessible on mobile and desktop

### Edge Cases
- [ ] Appointment cancelled by provider
- [ ] No-show status applied automatically
- [ ] Conflicting appointments detected
- [ ] Appointment moved to different provider/location

## Visual Design Context

### Screen References
- SCR-005: Appointment Calendar & History (main view)
- SCR-006: Appointment Detail (detail view)
- SCR-007: Reschedule Appointment (reschedule flow)

### Key UI Elements
- Calendar grid: Month/week/day views with appointment dots
- Appointment cards: Time, provider, location, status chip, actions menu
- Detail modal: Full appointment info, notes, map, contact buttons
- Action buttons: Reschedule, Cancel, Add to Calendar

### Design Tokens
- Status chips: Confirmed (green), Pending (amber), Cancelled (red)
- Calendar theme: Healthcare blue (#1F4E8D) for appointments
- Action buttons: Primary blue for reschedule, secondary gray for cancel

### Responsive Behavior
- Desktop: Full calendar with sidebar appointment list
- Tablet: Calendar with expandable appointment cards
- Mobile: List view with calendar toggle, swipe actions for reschedule/cancel

## Testing Notes

3. Selects reschedule, chooses new time
4. Confirms reschedule and receives notification
